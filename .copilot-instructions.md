**Last Updated:** 2026-06-03  
**Status:** Foundation document for interactive Copilot CLI workflow

## 1. Project Context

### Overview
**Mapper Copilot** is an AI-assisted RSC→SLCP question mapping suggester that uses a two-stage RAG (Retrieval-Augmented Generation) pipeline:
1. **Retrieve:** Semantic search (~800 SLCP questions) using embeddings → top-N candidates
2. **Reason:** LLM ranks candidates and drafts mapping rule fields (compliant_answer, match_type, etc.)

### Tech Stack
- **Runtime:** Python 3.10+ (currently running 3.9.6, compatible)
- **Backend:** FastAPI (secondary API) + Streamlit (primary UI)
- **Embeddings:** AWS Bedrock Titan Text Embeddings V2 (or mock HashingEmbedder for offline testing)
- **LLM:** AWS Bedrock Claude Sonnet (or mock MockLLM for offline testing)
- **Vector Search:** Numpy cosine similarity (POC) → FAISS/pgvector (future)
- **Excel Parsing:** openpyxl with configurable column mapping
- **Config Management:** Pydantic Settings (env-driven)
- **Testing:** pytest with offline mock providers (zero network dependency)

### Critical Constraint: Offline-First
All providers MUST support deterministic offline operation:
- `PROVIDER=mock` → uses HashingEmbedder + MockLLM (no network, no AWS, same output for same input)
- `PROVIDER=bedrock` → swaps to real AWS services (requires credentials, region, profile)
- **Tests must pass with mock providers** (CI/CD requirement)

## 2. Architecture & Design Patterns

### 2.1 Provider Abstraction Pattern

All external dependencies (embeddings, LLM, vector store) are **pluggable behind abstract interfaces**.

**Example: EmbeddingProvider**

```python
from abc import ABC, abstractmethod
import numpy as np

class EmbeddingProvider(ABC):
    """Abstract embedding provider."""
    
    @abstractmethod
    def embed(self, texts: List[str]) -> np.ndarray:
        """
        Embed a list of texts.
        
        Args:
            texts: List of strings to embed
            
        Returns:
            np.ndarray of shape (len(texts), embedding_dimension)
        """
        pass
```

**Implementations:**
- `HashingEmbedder(EmbeddingProvider)` — Deterministic token-hash embedding, offline, zero dependencies
- `BedrockEmbedder(EmbeddingProvider)` — AWS Bedrock Titan V2, real embeddings, requires AWS creds
- `SentenceTransformerEmbedder(EmbeddingProvider)` — Local transformers model, optional

**Factory Function:**
```python
def get_embedder(provider: str, **kwargs) -> EmbeddingProvider:
    """Instantiate embedder based on provider name."""
    if provider == "mock":
        return HashingEmbedder()
    elif provider == "bedrock":
        return BedrockEmbedder(kwargs.get("region"), kwargs.get("profile"))
    elif provider == "sentence_transformer":
        return SentenceTransformerEmbedder(kwargs.get("model_name"))
    else:
        raise ValueError(f"Unknown provider: {provider}")
```

**Same pattern applies to:**
- `LLMProvider` (MockLLM, BedrockLLM)
- `VectorStore` (NumpyVectorStore, FAISSVectorStore, pgvectorStore)

### 2.2 Two-Stage RAG Pipeline

**Stage 1: Retrieve**
```
RSC Question → Embed question text → Vector search in SLCP embeddings → Top-K candidates
```

**Stage 2: Reason**
```
RSC Question + Top-K SLCP candidates → LLM prompt → Rank + draft rule fields
```

**Orchestrator:** `Suggester` class coordinates both stages.

### 2.3 Error Handling & Logging

- **Custom exceptions:** Inherit from `Exception`, use descriptive messages
- **Logging:** Use Python `logging` module with named loggers (e.g., `logger = logging.getLogger(__name__)`)
- **Graceful degradation:** If AWS is unavailable, fall back to mock (only in test/dev mode)
- **No silent failures:** Always log warnings/errors before raising or returning None

## 5. Testing Patterns

### 5.1 Mock Providers (Offline, Deterministic)

**HashingEmbedder:** Deterministic token-hash embedding
```python
class HashingEmbedder(EmbeddingProvider):
    """Deterministic offline embedder for testing."""
    
    def embed(self, texts: List[str]) -> np.ndarray:
        """Hash-based embedding (same input → same output)."""
        embeddings = []
        for text in texts:
            # Deterministic hash → embedding
            hash_val = hash(text) % 512  # Fixed dimension
            embedding = np.random.RandomState(hash_val).normal(0, 1, 512)
            embeddings.append(embedding)
        return np.array(embeddings)
```

**MockLLM:** Lexical overlap heuristic ranking
```python
class MockLLM(LLMProvider):
    """Offline mock LLM for testing."""
    
    def complete(self, prompt: str) -> dict:
        """Return mock response (JSON with keys: matches, confidences, rationale)."""
        # Deterministic: extract SLCP keys from prompt, rank by lexical overlap
        return {
            "suggested_mappings": ["fp-oc-1"],
            "confidences": {"fp-oc-1": 0.85},
            "rationale": "Mock response: lexical match on 'license'",
            "draft_compliant_answer": "Yes",
            "draft_noncompliant_answer": "No",
            "draft_match_type": "EXACT",
        }
```

- Logging: Info for each embed call, debug for dimensions, warning for fallbacks

   - Use boto3 bedrock-runtime client

## 6.4 Step 6: Suggester (`src/mapper_copilot/core/suggester.py`)

**Dependencies:** All prior modules (embeddings, llm, vector_store, excel_loader, models)

**Purpose:** Orchestrate two-stage RAG pipeline: retrieve (vector search) → reason (LLM reasoning).

**Key Requirements:**
- Class `Suggester` with `__init__(embedder, llm_provider, vector_store, slcp_questions)`
- `suggest(rsc_question: RscQuestion, top_k: int = 5) -> MappingSuggestion` method
- Pipeline:
  1. Embed RSC question description
  2. Search vector store for top-K SLCP candidates
  3. Build prompt with RSC question + candidates
  4. Call LLM to rank & draft rule fields
  5. Return MappingSuggestion with rationale + draft fields
- Error handling: Fallback to top-K if LLM fails (graceful degradation)
- Logging: Info for each step, debug for vectors/prompts

**Prompt Template:**
```
Generate core/suggester.py with:
1. Suggester class
   - __init__(embedder: EmbeddingProvider, llm_provider: LLMProvider, vector_store: VectorStore, slcp_questions: List[SlcpQuestion])
   
2. suggest(rsc_question: RscQuestion, top_k: int = 5) -> MappingSuggestion method
   - Embed RSC question description
   - Search vector store for top_k SLCP candidates
   - Build prompt: "Rank these SLCP questions as potential matches for this RSC question: <rsc_text>"
   - Call llm_provider.complete(prompt)
   - Parse response (suggested_mappings, confidences, rationale, draft_* fields)
   - Return MappingSuggestion with timestamp
   
3. Error handling: graceful fallback if LLM fails

4. Full type hints, docstrings, logging
```

## 6.5 Step 7: Evaluation Harness (`src/mapper_copilot/eval/harness.py`)

**Dependencies:** `models.py`, `suggester.py`

**Purpose:** Batch evaluation of suggestions against ground-truth mappings.

**Key Requirements:**
- Class `EvaluationHarness` with `__init__(suggester: Suggester, ground_truth_mappings: Dict[str, str])`
- `evaluate(rsc_questions: List[RscQuestion]) -> List[EvaluationResult]` method
- Metrics: Hit@1, Hit@5, Precision@5, MRR (Mean Reciprocal Rank)
- Output: Summary stats + per-question results
- Logging: Progress bar (or info logs) during evaluation

**Prompt Template:**
```
Generate eval/harness.py with:
1. EvaluationHarness class
   - __init__(suggester: Suggester, ground_truth_mappings: Dict[str, str])
   - ground_truth_mappings: {rsc_question_id → slcp_key}

2. evaluate(rsc_questions: List[RscQuestion]) -> List[EvaluationResult] method
   - For each RSC question:
     - Get suggestion from suggester.suggest()
     - Compare suggested_mappings vs ground_truth[rsc_id]
     - Create EvaluationResult with hit=true/false, rank_if_hit
   - Return all results

3. compute_metrics(results: List[EvaluationResult]) -> dict method
   - Hit@1: % of results with rank_if_hit==1
   - Hit@5: % of results with rank_if_hit<=5
   - Precision@5: precision@5 metric
   - MRR: mean(1/rank_if_hit for hits)
   - Return dict with all metrics

4. Full type hints, docstrings, logging
```
